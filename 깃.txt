1. 깃허브란?
깃을 사용하는 프로젝트를 지원하는 웹 기반의 호스팅 서비스이다. 

GitHub는 영리적인 서비스와 오픈소스를 위한 무상 서비스를 모두 제공한다.

SVN,TEMF,Source Safe 같은
   가장 인기있는 오픈 소스 코드 저장소임. 2222


깃허브 는 깃을 사용하는 프로젝트를 지원하는 웹 기반의 호스팅 서비스이다. 루비 온 레일스로 작성되었다.
 GitHub는 영리적인 서비스와 오픈소스를 위한 무상 서비스를 모두 제공한다. 
2009년의 Git 사용자 조사에 따르면 GitHub는 가장 인기있는 Git 호스팅 사이트이다. 
또한 2011년의 조사에서는 가장 인기있는 오픈 소스 코드 저장소로 꼽혔다.
깃허브는 pastebin과 유사한 서비스와 위키를 각 저장소마다 운영하고 있으며 깃 저장소를 통해 고칠 수 있다.
2010년 1월부터 깃허브는 GitHub, Inc. 라는 이름으로 운영되고 있다.

((웹 이라는 부분이 장점이라 뭔짓이든 할수있을꺼같은데?)

2. 깃허브를 통해 할 수 있는 것들. 
 - 특정 개발프로그램을 가리는것이 아님 그냥 어떤 개발엔진에서 개발했다는 표시만 문서거 뭐고 죄다 올라감
 - 다른사람의 소스를 레파지토리 단위로 통채로 내 계정에 복사할수있다
  (그냥 복사한다기보다 내 계정에 하나의 레파지토리를 만든다 fork라 부름) fork시 내 계정으로 바로 생성되어 사용가능
   (문서까지 다됨) 소스관리 하나 끝내줌
  - 해당 레파지토리 소스에 관해 지트 유저들이 오류정정과 코맨트 업데이트 
  - 웹 내에서 소스 직접 수정 : 소스 다운후 수정후 싱크 맞추기
  - 검색된 단어중 어떤 프로그램으로 개발된 소스들의 카운트 확인가능(EX,자바로 된거 몇개.C로 된거 몇개)
  - 개인 메모 장소로도 사용됨 (물론 검색어에 걸림)
  - 개발자 아이디도 포함됨 
  - 공개 비공개 소스로 가능(비공개는 기업용이겠지)
 -  자신이 얼마나 소스를 공개했으며 언제 많이 몇개를 만들어올렸고 
  -.csproj sin  등은 실행시 자동생성되어 폴더를 공유하게 되는데 gitignore 로 커밋이 이루어지지 않는 파일로 세팅 가능
  - 일단 오픈이며 월 정액제로 레파지토리 돈을 받고있음
  - 월사용료 방식으로 돈을 받고있음(프라이빗 저장소 값)
  - md해외판매 사이트와 이메일 인증방식이 같음(메일링 확인)
  - 개인이 올린 소스에관해 그래프로 가시화됨 + 트위터+페북짬뽕같이 같이 팔로잉 팔 로워 좋아요 기능이있음 
  -  소스관리에 브랜치라는게 핵심인데 일단 패쓰
  - 이슈리스트의 질문 답변은 최초 글이 질문이며 나머지는 모두 답변으로 카운트됨
